# Пошаговый запуск Digital Brain (инструкция для новичка)

Ниже — максимально простой путь: от «ничего не настроено» до «бот работает и я им пользуюсь».

---

## 0) Что ты в итоге получишь

После выполнения шагов у тебя будет Telegram-бот, который:

1. Принимает текстовые сообщения.
2. Принимает голосовые (если указан API-ключ OpenAI для транскрибации).
3. Сохраняет всё в локальные Markdown-файлы в папке `data/`.
4. Позволяет искать по сохранённому через `/ask`.

---

## 1) Что нужно подготовить заранее

### 1.1 Установи Git
**Git** — инструмент, который скачивает и обновляет код проекта.

Проверка:

```bash
git --version
```

### 1.2 Установи Python 3.11+
**Python** — язык, на котором написан бот.

Проверка:

```bash
python3 --version
```

Если версия ниже 3.11 — обнови Python.

---

## 2) Скачай проект на компьютер

```bash
git clone <URL_твоего_репозитория>
cd <папка_репозитория>
```

---

## 3) Создай Telegram-бота (через BotFather)

1. Открой в Telegram: `@BotFather`.
2. Отправь команду `/newbot`.
3. Придумай имя и username для бота.
4. BotFather пришлёт **токен** (длинная строка вида `12345:ABC...`).

**Токен** — это пароль бота. Никому его не отправляй.

---

## 4) Подготовь проект одной командой

```bash
./scripts/bootstrap.sh
```

Что это делает:
- создаёт виртуальное окружение `.venv`;
- ставит зависимости;
- создаёт файл `.env` (если его ещё нет).

**Виртуальное окружение** — изолированная папка с библиотеками проекта.

---

## 5) Заполни `.env`

Открой файл `.env` и укажи:

```env
TELEGRAM_BOT_TOKEN=вставь_сюда_токен_из_BotFather
OPENAI_API_KEY=опционально_для_голосовых
DATA_DIR=./data
```

- `TELEGRAM_BOT_TOKEN` — обязательно.
- `OPENAI_API_KEY` — нужен только для распознавания голосовых.

---

## 6) Запусти бота

```bash
./scripts/run_local.sh
```

Если всё нормально, увидишь сообщение в консоли, что бот запущен.

---

## 7) Проверь, что работает

Открой своего бота в Telegram и отправь:

1. `/start`
2. `/save тестовая заметка`
3. `/ask тестовая`
4. `/review daily`

Проверь файлы:

- `data/inbox/` — заметки;
- `data/reviews/` — шаблоны review;
- `data/logs/operations.md` — журнал операций.

---

## 8) Как пользоваться каждый день (минимальный ритм)

### Daily (каждый день)
1. Наговорил мысль/задачу в Telegram.
2. Вечером: `/review daily`.
3. Допиши 3 пункта: Wins / Problems / Next Actions.

### Weekly (раз в неделю)
1. `/review weekly`
2. Просмотри `data/inbox/` и выдели главное.
3. Сформулируй план на следующую неделю.

---

## 9) Что делать, если не запускается

### Ошибка: `TELEGRAM_BOT_TOKEN is empty`
Проверь `.env`, поле `TELEGRAM_BOT_TOKEN` должно быть заполнено.

### Ошибка: не распознаются голосовые
Скорее всего, не заполнен `OPENAI_API_KEY`.

### Ошибка: `.venv not found`
Сначала выполни:

```bash
./scripts/bootstrap.sh
```

---

## 10) Когда переносить в «прод» (production)

**Production** = «работает постоянно, не только когда открыт твой ноутбук».

Переходи в прод, когда локально стабильно работает 5–7 дней подряд.

Минимальный план переноса:
1. Выбрать сервер (VPS/облако).
2. Скопировать репозиторий на сервер.
3. Настроить `.env` на сервере.
4. Запускать бота как фоновый сервис (например, `systemd` или контейнер).
5. Настроить бэкапы папки `data/`.

---

## 11) Чеклист «готово к работе»

- [ ] Бот создан в BotFather.
- [ ] `.env` заполнен.
- [ ] Команда `./scripts/run_local.sh` запускается без ошибок.
- [ ] `/save` создаёт файлы в `data/inbox/`.
- [ ] `/ask` возвращает результаты.
- [ ] `/review daily` создаёт файл review.

Если все пункты отмечены — у тебя уже рабочее решение MVP.
