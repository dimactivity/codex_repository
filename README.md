# Digital Brain MVP (Telegram + Markdown)

Этот репозиторий содержит MVP твоего «digital brain»:

- вход: **текст и голос** через Telegram;
- сохранение: в **Markdown-хранилище**;
- извлечение: простые запросы в Telegram (`/ask ...`) по локальному контексту;
- журнал операций: в Markdown (`data/logs/operations.md`);
- режимы: автоопределение intent (Capture / Command / Coach / Review), при сомнении — уточнение.

## Почему такая архитектура

- **Telegram**: готовый, стабильный UI с iPhone/Mac, голос и текст из коробки.
- **Markdown**: прозрачное и переносимое хранилище, удобно читать вручную и версионировать через git.
- **Без БД в MVP**: проще запуск и сопровождение; при росте можно добавить SQLite без миграции UX.

## Структура

- `src/digital_brain_bot.py` — Telegram-бот и orchestration.
- `src/storage.py` — сохранение/поиск/журнал операций в Markdown.
- `src/intents.py` — базовое определение интентов и разбор мульти-интент запросов.
- `data/` — хранилище Markdown.
  - `inbox/` — входящие capture-записи.
  - `entities/` — структурированные сущности (пока простая раскладка).
  - `reviews/` — daily/weekly записи.
  - `logs/operations.md` — журнал действий.
- `.env.example` — переменные окружения.
- `requirements.txt` — зависимости.

## Быстрый старт

1. Создай бота в Telegram через `@BotFather`, получи токен.
2. Скопируй `.env.example` в `.env` и заполни:
   - `TELEGRAM_BOT_TOKEN`
   - `OPENAI_API_KEY` (опционально, для транскрибации голосовых)
3. Установи зависимости:

```bash
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

4. Запусти:

```bash
python src/digital_brain_bot.py
```

## Команды в Telegram

- `/start` — приветствие и краткая помощь.
- `/save <текст>` — сохранить явно.
- `/ask <вопрос>` — извлечь информацию из хранилища.
- `/review daily` / `/review weekly` — создать review-шаблон.

Также можно просто отправлять текст — бот сам определит intent.

## Логика подтверждений

- Сохранение новой информации — **без лишнего подтверждения**.
- Массовые изменения/реорганизация — бот просит подтверждение.
- Удаление — всегда требует явного подтверждения `YES_DELETE`.

## Ограничения MVP

- Голос: пока транскрибация + сохранение (без автоизвлечения задач/решений).
- Поиск: простая текстовая релевантность по markdown-файлам.
- Дашборды/UI — отложены после валидации базового workflow.
